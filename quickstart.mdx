## Quick Start

This guide will help you record your first video with Cap in just a few minutes.

<Steps>
  <Step title="Choose Your Platform">
    Cap offers two ways to get started:
    
    - **Cap Desktop** - Native app for macOS and Windows (recommended for recording)
    - **Cap Web** - Browser-based platform for uploading, sharing, and viewing videos
    
    Most users start with the desktop app to record videos.
  </Step>
  
  <Step title="Install Cap Desktop">
    Download and install the Cap Desktop app for your operating system:
    
    <Card title="Download Cap Desktop" icon="download" href="https://cap.so/download">
      Download for macOS or Windows
    </Card>
    
    **System Requirements:**
    - macOS 11+ or Windows 10+
    - 4GB RAM minimum (8GB recommended)
    - 500MB free disk space
    
    See the [Desktop Installation Guide](/installation/desktop) for detailed instructions.
  </Step>
  
  <Step title="Set Up Permissions">
    On first launch, Cap will request permissions:
    
    **macOS:**
    - Screen Recording permission
    - Microphone access (if recording audio)
    - Camera access (if using webcam)
    
    **Windows:**
    - Windows will prompt for necessary permissions
    - Grant access when prompted
    
    <Note>
      On macOS, if running from terminal during development, grant permissions to the terminal app instead of Cap.
    </Note>
  </Step>
  
  <Step title="Configure Recording">
    Choose your recording options:
    
    1. **Recording Mode:**
       - Full screen
       - Specific window
       - Custom area
       - Camera only
    
    2. **Audio Settings:**
       - System audio
       - Microphone
       - Both
       - None
    
    3. **Camera Settings:**
       - Enable/disable webcam
       - Choose camera device
       - Select preview shape and size
  </Step>
  
  <Step title="Start Recording">
    Click the **Record** button to start:
    
    1. A countdown will begin (3 seconds by default)
    2. Recording starts automatically
    3. Use the controls to:
       - Pause/resume recording
       - Stop recording
       - Cancel recording
    
    <Tip>
      Use keyboard shortcuts for quick control. Check Settings → Hotkeys for available shortcuts.
    </Tip>
  </Step>
  
  <Step title="Edit Your Video (Optional)">
    After recording in Studio Mode:
    
    - Trim the beginning and end
    - Add visual effects (blur, backgrounds)
    - Add text overlays
    - Configure export settings
    
    See the [Video Editing Guide](/desktop/editing) for details.
  </Step>
  
  <Step title="Share Your Recording">
    Choose how to share:
    
    **Upload to Cap Web:**
    - Automatic upload after recording (default)
    - Get a shareable link instantly
    - Track views and engagement
    
    **Export Locally:**
    - Save as MP4 file
    - Use for other platforms
    - No upload required
    
    <CardGroup cols={2}>
      <Card title="Sharing Guide" icon="share" href="/web/sharing">
        Learn about sharing options
      </Card>
      <Card title="Analytics" icon="chart-line" href="/web/analytics">
        Track video performance
      </Card>
    </CardGroup>
  </Step>
</Steps>

## Recording Locations

Your recordings are stored locally before upload:

- **macOS**: `~/Library/Application Support/so.cap.desktop.dev/recordings`
- **Windows**: `%programfiles%/so.cap.desktop.dev/recordings`

## Common Issues

<AccordionGroup>
  <Accordion title="Permissions not working on macOS">
    If recording doesn't work after granting permissions:
    
    1. Go to System Settings → Privacy & Security
    2. Find Screen Recording
    3. Ensure Cap is checked
    4. Restart Cap Desktop
  </Accordion>
  
  <Accordion title="No audio in recordings">
    Make sure to:
    - Enable system audio in recording settings
    - Check audio device selection
    - Test audio levels before recording
    - Ensure apps you're recording have audio output
  </Accordion>
  
  <Accordion title="Video upload failing">
    If uploads fail:
    - Check your internet connection
    - Verify Cap server URL in Settings
    - Check server logs if self-hosting
    - Try exporting locally as backup
  </Accordion>
  
  <Accordion title="Recording is laggy">
    To improve performance:
    - Close unnecessary applications
    - Reduce recording resolution if available
    - Ensure sufficient disk space
    - Check system resource usage
  </Accordion>
</AccordionGroup>
